# 1. Концепции программных проектов

### 1\. TaskFlow — персональный менеджер задач

Описание:
Приложение для управления личными задачами с фокусом на приоритеты и дедлайны. Пользователь может создавать задачи, группировать их по проектам, задавать приоритеты и отслеживать выполнение.

Юзкейсы:
Создать задачу, отредактировать задачу, удалить задачу, отметить выполненной, задать дедлайн, изменить приоритет, сгруппировать по проекту, просмотреть список на сегодня, архивировать задачи.

### 2\. StudyBuddy — учебный трекер

Описание:
Приложение для студентов, позволяющее отслеживать учебные активности: пары, дедлайны, домашние задания и подготовку к экзаменам.

Юзкейсы:
Добавить предмет, добавить дедлайн, отметить выполненное задание, просмотреть расписание, получить напоминание, просмотреть статистику по предмету, редактировать предмет, удалить предмет.

### 3\. EventPulse — планировщик мероприятий

Описание:
Сервис для организации небольших мероприятий (дни рождения, встречи, митапы). Пользователь создает событие, участники выбирают предпочтения, система помогает выбрать оптимальный вариант.

Юзкейсы:
Создать событие, пригласить участников, выбрать дату, выбрать формат, оставить комментарий, просмотреть результаты, подтвердить событие, отменить событие, экспортировать событие.

### 4\. BudgetLite — простой контроль бюджета

Описание:
Минималистичное приложение для учета личных расходов и доходов с акцентом на простоту и скорость ввода данных.

Юзкейсы:
Добавить расход, добавить доход, выбрать категорию, удалить запись, просмотреть баланс, фильтровать по периоду, просмотреть статистику, установить лимит.

# 2. Критерии для оценки концепций

C1: Ценность для пользователя – Насколько готовый MVP будет решать конкретные проблемы или удовлетворять потребности целевой аудитории. Оценивает полезность и востребованность основного функционала продукта с точки зрения конечного пользователя.

C2: Техническая сложность – Оценка объема и сложности разработки, включая требуемые технологии, архитектурные решения, интеграции и потенциальные технические риски. Критерий-издержка, где меньшее значение указывает на более реализуемый проект в заданных ограничениях.

C3: Потенциал развития – Способность проекта расширяться функционально и технически после реализации MVP. Включает как рыночный потенциал (аудитория, монетизация), так и архитектурную возможность добавлять новые модули и масштабировать систему.

C4: Интерес к реализации – Уровень вовлеченности и энтузиазма команды в работе над конкретным проектом. Включает личную заинтересованность участников, желание развивать навыки через эту задачу и командную синергию. Критически важен для долгосрочной продуктивности в учебном контексте.
