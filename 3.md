# Фаза 3. Оценка проекта

## 1. Сценарии использования

Акторы системы: Студент, система уведомлений, база данных.

### UC-1. Добавить предмет
*Акторы*: Студент\
*Цель*: добавить новый учебный предмет для дальнейшего планирования заданий\
*Предусловия*: Студент авторизован в системе\
*Основной поток*:
1. Студент нажимает кнопку «Добавить предмет».
2. Студент вводит название предмета.
3. Студент (опционально) вводит описание предмета.
4. Студент нажимает кнопку «Сохранить».
5. Система сохраняет предмет и отображает его в списке.

*Негативные*:
1. **Пустое название предмета**\
После нажатия кнопки «Сохранить» Система выводит сообщение об ошибке и предлагает заполнить поле.

*Постусловия*: Предмет сохранён в системе и доступен для добавления заданий.

### UC-2. Редактировать предмет
*Акторы*: Студент\
*Цель*: изменить данные существующего предмета\
*Предусловия*: В системе существует хотя бы один предмет\
*Основной поток*:
1. Студент выбирает предмет из списка.
2. Студент нажимает кнопку «Редактировать».
3. Студент изменяет название или описание.
4. Студент нажимает кнопку «Сохранить».
5. Система обновляет данные предмета.

*Альтернативные*:
1. **Отмена редактирования**\
Студент нажимает кнопку «Отменить», изменения не сохраняются.

*Постусловия*: Данные предмета обновлены.

### UC-3. Удалить предмет
*Акторы*: Студент\
*Цель*: удалить предмет и связанные с ним задания\
*Предусловия*: В системе существует предмет\
*Основной поток*:
1. Студент выбирает предмет.
2. Студент нажимает кнопку «Удалить».
3. Система запрашивает подтверждение.
4. Студент подтверждает удаление.
5. Система удаляет предмет и связанные задания.

*Негативные*:
1. **Отмена удаления**\
Студент отклоняет подтверждение, предмет остаётся в системе.

*Постусловия*: Предмет и связанные задания удалены.

### UC-4. Добавить учебное задание
*Акторы*: Студент\
*Цель*: создать задание по предмету с дедлайном\
*Предусловия*: В системе существует хотя бы один предмет\
*Основной поток*:
1. Студент нажимает кнопку «Добавить задание».
2. Студент выбирает предмет.
3. Студент вводит название задания.
4. Студент указывает дедлайн.
5. Студент нажимает кнопку «Сохранить».
6. Система сохраняет задание.

*Негативные*:
1. **Дедлайн в прошлом**\
После нажатия кнопки «Сохранить» Система выводит сообщение об ошибке.

*Постусловия*: Задание сохранено и отображается в списке заданий.

### UC-5. Отметить задание выполненным
*Акторы*: Студент\
*Цель*: зафиксировать выполнение задания\
*Предусловия*: Существует активное задание\
*Основной поток*:
1. Студент открывает список заданий.
2. Студент выбирает задание.
3. Студент отмечает его как выполненное.
4. Система обновляет статус задания.

*Постусловия*: Задание отмечено как выполненное.

### UC-6. Просмотреть список заданий
*Акторы*: Студент\
*Цель*: получить обзор всех заданий\
*Предусловия*: Пользователь авторизован\
*Основной поток*:
1. Студент открывает раздел «Задания».
2. Система загружает задания из базы данных.
3. Система отображает список заданий.

*Альтернативные*:
1. **Фильтрация заданий**\
Студент выбирает фильтр (по предмету или дедлайну), после чего Система обновляет отображаемый список.

*Постусловия*: Студент видит актуальный список заданий.

### UC-7. Просмотреть расписание
*Акторы*: Студент\
*Цель*: просмотреть учебное расписание\
*Предусловия*: Расписание задано в системе\
*Основной поток*:
1. Студент открывает раздел «Расписание».
2. Система загружает данные расписания.
3. Система отображает занятия по дням.

*Негативные*:
1. **Расписание отсутствует**\
Система отображает информационное сообщение.

*Постусловия*: Расписание отображено пользователю.

### UC-8. Установить напоминание о дедлайне
*Акторы*: Студент, Система уведомлений\
*Цель*: получить напоминание перед дедлайном задания\
*Предусловия*: Задание существует\
*Основной поток*:
1. Студент выбирает задание.
2. Студент нажимает «Добавить напоминание».
3. Студент указывает дату и время напоминания.
4. Студент сохраняет настройки.
5. Система планирует уведомление.

*Негативные*:
1. **Время напоминания позже дедлайна**\
После сохранения настроек Система выводит предупреждение.

*Постусловия*: Напоминание сохранено.

### UC-9. Просмотреть статистику по предмету
*Акторы*: Студент\
*Цель*: оценить прогресс по предмету\
*Предусловия*: По предмету существуют задания\
*Основной поток*:
1. Студент выбирает предмет.
2. Студент открывает раздел «Статистика».
3. Система анализирует выполненные и невыполненные задания.
4. Система отображает статистику.

*Альтернативные*:
1. **Отсутствие данных**\
Система отображает сообщение «Недостаточно данных».

*Постусловия*: Статистика отображена пользователю.

### UC-10. Получить уведомление о дедлайне
*Акторы*: Система уведомлений\
*Цель*: уведомить студента о приближающемся дедлайне\
*Предусловия*: Для задания настроено напоминание\
*Основной поток*:
1. Система проверяет задания с активными напоминаниями.
2. Система формирует уведомление.
3. Система отправляет уведомление студенту.

*Постусловия*: Студент получает уведомление о дедлайне.
